Learn about what Knative is and if you should get involved.

{{< /blocks/cover >}} {{% blocks/lead %}} Knative (pronounced kay-nay-tiv) extends Kubernetes to provide a set of middleware components that are essential to build modern, source-centric, and container-based applications that can run anywhere: on premises, in the cloud, or even in a third-party data center. The Knative project attempts to identify common patterns and codify the best practices that are shared by successful real-world Kubernetes-based frameworks and applications. Developers on Knative can use familiar idioms, languages, and frameworks to deploy any workload: functions, applications, or containers. {{% /blocks/lead %}} {{< blocks/section >}}

Build and use, what you want

Install and use individual Knative components:
  • Build - Source-to-container build orchestration
  • Eventing - Management and delivery of events
  • Serving - Request-driven compute that can scale to zero

Solve your everyday tasks

  • Deploying a container
  • Orchestrating source-to-URL workflows on Kubernetes
  • Routing and managing traffic with blue/green deployment
  • Automatic scaling and sizing workloads based on demand
  • Binding running services to eventing ecosystems
